## 2. Top Credits For Full Automation (~4000 words, 8 detailed blueprints)
### 2.1 IPp1 Climate Resilience Assessment
#### 2.1.1 Why this credit is ideal for automation: structured hazard assessment with public data APIs
#### 2.1.2 Required consultant inputs: project address, building type, service life projection
#### 2.1.3 AI workflow: IPCC data retrieval, FEMA/USGS/NOAA API integration, hazard scoring engine, narrative generation
#### 2.1.4 Generated outputs: completed climate resilience assessment report with priority hazard analysis
### 2.2 IPp2 Human Impact Assessment
#### 2.2.1 Automation rationale: demographic and infrastructure data fully available through public APIs
#### 2.2.2 Required inputs: project address, occupancy type, vulnerable population flags
#### 2.2.3 AI workflow: Census data retrieval, Walk Score API, infrastructure mapping, four-category assessment generation
#### 2.2.4 Generated outputs: complete human impact assessment with demographics, infrastructure, health impacts, occupant experience
### 2.3 IPc2 Green Leases
#### 2.3.1 Automation rationale: template-driven document with 18 best practices from LEED reference guide
#### 2.3.2 Required inputs: base building LEED attempted credits list, tenant space types, owner requirements
#### 2.3.3 AI workflow: RAG over LEED v5 requirements, best practice matching, lease clause generation, scoring optimization
#### 2.3.4 Generated outputs: standard green lease document with compliance checklist and scoring table
### 2.4 WEp2 Minimum Water Efficiency and WEc2 Enhanced Water Efficiency
#### 2.4.1 Unified automation pipeline: fixture schedule ingestion to complete water calculation and documentation
#### 2.4.2 Required inputs: fixture schedule (uploaded), project occupancy, building type
#### 2.4.3 AI workflow: OCR/structured extraction of fixture flows, baseline calculation engine, reduction percentage optimization, full narrative generation
#### 2.4.4 Generated outputs: water use baseline table, reduction calculation, fixture compliance matrix, narrative
### 2.5 EQp1 Construction Management and EQp2 Fundamental Air Quality
#### 2.5.1 Automation rationale: highly templated plans following ASHRAE 62.1 and standard construction practice
#### 2.5.2 Required inputs: project type, IAQ specifications, construction schedule, MERV filter selections
#### 2.5.3 AI workflow: template selection, project-specific parameter insertion, ASHRAE compliance checking, plan generation
#### 2.5.4 Generated outputs: Construction Activity Pollution Prevention plan, Indoor Air Quality Management Plan
### 2.6 MRc3 Low-emitting Materials
#### 2.6.1 Automation rationale: pure rule-based compliance engine against CDPH/BIFMA/CARB thresholds
#### 2.6.2 Required inputs: product list with manufacturer data, EPDs, test reports (uploaded)
#### 2.6.3 AI workflow: EPD parsing, certification database lookup, threshold comparison, compliance table generation
#### 2.6.4 Generated outputs: material compliance matrix, VOC threshold comparison table, non-compliant item flagging
### 2.7 LTc3 Compact and Connected Development
#### 2.7.1 Automation rationale: 100% public data-driven through Walk Score, transit APIs, Census data
#### 2.7.2 Required inputs: project address only — all other data fetched automatically
#### 2.7.3 AI workflow: multi-API data retrieval (Walk Score, GTFS, Census, Google Places), threshold checking, points optimization
#### 2.7.4 Generated outputs: connectivity analysis report, density calculation, transit access documentation
### 2.8 EAc7 Enhanced Refrigerant Management and EAp5 Fundamental Refrigerant Management
#### 2.8.1 Automation rationale: pure database and calculation work with standardized GWP benchmarks
#### 2.8.2 Required inputs: equipment schedule with refrigerant types and quantities (uploaded)
#### 2.8.3 AI workflow: refrigerant inventory parsing, GWP database lookup, weighted average calculation, leak detection analysis
#### 2.8.4 Generated outputs: complete refrigerant inventory, GWP analysis table, compliance declaration

## 4. Credits To Avoid In The First Version (~1500 words)
### 4.1 Energy Modeling-Dependent Credits
#### 4.1.1 EAp2 Minimum Energy Efficiency, EAc2 Reduce Peak Thermal Loads, EAc3 Enhanced Energy Efficiency: require ASHRAE 90.1 Appendix G modeling in eQuest/EnergyPlus/TRACE
#### 4.1.2 Why automation is impractical: model calibration, building geometry creation, system simulation require specialized engineering software and judgment
### 4.2 Physical Verification and Testing Credits
#### 4.2.1 EQc5 Air Quality Testing and Monitoring: requires physical laboratory testing of IAQ parameters
#### 4.2.2 SSp1 Minimized Site Disturbance: requires site verification and photographic documentation of actual conditions
### 4.3 Complex Commissioning and Operational Credits
#### 4.3.1 EAp3 Fundamental Commissioning and EAc5 Enhanced Commissioning: require CxA engagement, field verification, functional testing
#### 4.3.2 EAc6 Grid Interactive: requires utility coordination, actual storage system specification, DR program enrollment
### 4.4 Credits with Limited Repeatability
#### 4.4.1 PRc1 Project Priorities: highly project-specific, requires custom analysis for each project location and type
#### 4.4.2 MRc1 Building and Materials Reuse: depends on existing building conditions, requires field verification of salvageable materials
