## Executive Summary

The EA category presents a **mixed automation landscape** with significant opportunities for AI-driven document generation, carbon calculation engines, and refrigerant database lookups, alongside inherently manual engineering simulation work that cannot be automated. The highest-value automation targets are narrative-based documents (decarbonization plans, commissioning plans), refrigerant inventory calculations, and electrification COP calculations -- all of which are highly templatable with clear input/output relationships.

### EAp1: Operational Carbon Projection and Decarbonization Plan

| Attribute | Detail |
|-----------|--------|
| **Automation Score** | 4 / 5 |
| **Commercial Value** | 5 / 5 |
| **Risk Level** | Low |
| **Documentation Type** | Narrative + Plan + Attestation |
| **Recommendation** | **Automate in MVP** |

#### Why This Score
The decarbonization plan is a **narrative document capped at 2 pages** that describes retrofits, timelines, costs, avoided costs, and electrification readiness strategies. This is fundamentally a templatable document that AI can generate from structured project inputs. The site energy estimates come from energy model outputs that AI can parse and format for USGBC submission. The owner attestation is a simple signature document.

**The one element AI cannot generate:** The USGBC BAU carbon projection (this is generated by USGBC from submitted data). However, for BPS-subject projects, AI can help calculate ordinance-specific overlays, cap comparisons, and cumulative fine projections.

#### Required Inputs

| Input Source | What Energy Modeler Provides | What AI Generates |
|-------------|------------------------------|-------------------|
| Energy Model Outputs | Annual site energy by fuel type (kWh, therms, etc.) | Formatted USGBC submission data |
| Project Data | Project location (climate zone, grid region) | Decarbonization plan narrative |
| Design Data | Electrification readiness measures installed | Timeline of retrofit measures with cost estimates |
| BPS Data (if applicable) | Local ordinance carbon caps | BAU overlay with cap comparison, fine calculations |
| Owner | Signature/attestation | Pre-filled attestation document |

#### AI Techniques Applicable

1. **Energy model output parsing and interpretation** -- Parse eQuest, EnergyPlus, IES-VE, or Trane TRACE outputs to extract annual site energy by fuel type
2. **Carbon calculation engines** -- Calculate ordinance-specific carbon projections using local grid emission factors
3. **Narrative generation** -- Generate the 2-page decarbonization plan from structured project data, including retrofit timelines, cost estimates, avoided cost calculations, and electrification readiness strategies
4. **Document templating** -- Generate owner attestation forms pre-filled with project data

#### Automation Blueprint

```
Step 1: INPUT PARSING
  - Parse energy model output files (eQuest .SIM, EnergyPlus .HTML, TRACE 700)
  - Extract: Annual electricity (kWh), Annual gas (therms), Annual other fuels
  - Validate: Total energy matches expected ranges for building type/size
  - Cross-reference with ASHRAE 90.1 baseline if available

Step 2: SITE ENERGY FORMATTING
  - Format data per USGBC submission requirements
  - Convert all units to consistent format (kWh, therms, kBtu)
  - Generate CSV/JSON submission package

Step 3: DECARBONIZATION PLAN GENERATION
  INPUTS NEEDED FROM USER:
    - Electrification readiness measures (checklist)
    - Project type (office, healthcare, multifamily, etc.)
    - Heating/cooling system types
    - Planned vs. future equipment
    - Budget constraints (optional)
  
  AI GENERATES:
    - Section 1: Current design analysis summary
    - Section 2: Proposed retrofit measures with approximate timeline
    - Section 3: Cost estimates for each retrofit measure
    - Section 4: Equipment discard/replacement descriptions
    - Section 5: Electrification readiness strategies with avoided cost calculations
    - Section 6: Core and Shell tenant build-out considerations (if applicable)
    - Ensure narrative stays under 2-page limit

Step 4: BPS OVERLAY (if applicable)
  - Lookup local BPS ordinance requirements
  - Apply carbon caps to BAU projection
  - Calculate annual and cumulative fines/fees
  - Generate overlay visualization

Step 5: ATTESTATION DOCUMENT
  - Generate pre-filled attestation form
  - Flag for owner signature

Step 6: QUALITY REVIEW
  - Check narrative against LEED requirements checklist
  - Verify all required elements included
  - Flag for professional review
```

#### Integration Points
- **Input:** Energy model output files (eQuest, EnergyPlus, TRACE, IES-VE)
- **Input:** Project location data (zip code for climate zone + grid region lookup)
- **Input:** Equipment schedules (for electrification readiness inventory)
- **Output:** USGBC site energy submission package
- **Output:** Decarbonization plan narrative (DOCX/PDF)
- **Output:** Owner attestation form (PDF)

#### Estimated Time Savings
- Manual process: 8-16 hours (energy data compilation + narrative writing + formatting)
- Automated process: 1-2 hours (review + minor edits)
- **Time savings: 80-90%**

### EAp2: Minimum Energy Efficiency

| Attribute | Detail |
|-----------|--------|
| **Automation Score** | 2 / 5 |
| **Commercial Value** | 5 / 5 |
| **Risk Level** | Medium |
| **Documentation Type** | Energy Model + Calculation |
| **Recommendation** | **Assist Only** |

#### Why This Score
The core of EAp2 is energy modeling per ASHRAE 90.1 Appendix G. **Energy modeling itself cannot be automated** -- it requires specialized software (eQuest, EnergyPlus, TRACE, IES-VE) operated by licensed energy modelers who make hundreds of design-specific decisions. The model must be built, calibrated, and run by humans.

However, AI can significantly assist **after** the model is complete:
- Parse model outputs to extract regulated/unregulated energy
- Calculate Performance Index (PI) using future source energy
- Apply correct Building Performance Factors (BPFs) from climate zone tables
- Generate compliance documentation from model results
- Create comparison tables and summary reports

#### Required Inputs

| Input Source | What Energy Modeler Provides | What AI Can Assist |
|-------------|------------------------------|-------------------|
| Energy Model | Complete ASHRAE 90.1 Appendix G model | Parse outputs, extract regulated/unregulated energy |
| Modeler | Baseline building performance (BBP) | Calculate PI with future source energy |
| Modeler | Proposed building performance (PBP) | Apply correct BPF from climate zone table |
| Project Info | Climate zone, building type, ASHRAE version | Generate compliance summary report |
| Modeler | Model QA/QC, calibration | Flag unusual results for review |

#### AI Techniques Applicable

1. **Energy model output parsing and interpretation** -- Extract kBtu/sf, EUI, end-use breakdowns from model reports
2. **Compliance checking against ASHRAE 90.1** -- Verify that model meets required inputs/outputs per Appendix G
3. **BPF table lookup automation** -- Automatic climate zone to BPF mapping
4. **Calculation automation** -- PI = PBP/BBP, future source energy = site energy x 2.0
5. **Documentation generation** -- Summary reports, comparison tables

#### Limitations
- AI cannot BUILD energy models
- AI cannot calibrate models
- AI cannot make design decisions about system selections
- AI cannot perform quality assurance on model inputs
- All automated outputs must be reviewed by the energy modeler

#### Automation Blueprint (Assist Mode)

```
Step 1: MODEL OUTPUT INGESTION
  - Accept energy model output files
  - Parse: Baseline building energy, Proposed building energy
  - Parse: End-use breakdowns, fuel types, peak demands
  - Store in structured format

Step 2: BPF AUTOMATED LOOKUP
  - Input: Climate zone (from project zip code)
  - Input: Building type (from project data)
  - Input: ASHRAE version (90.1-2019 or 90.1-2022)
  - Output: Correct BPF value from published tables

Step 3: PI CALCULATION
  - Apply future source energy conversion (site x 2.0)
  - Calculate PI = PBP / BBP
  - Compare PI to BPF target
  - Determine pass/fail

Step 4: COMPLIANCE DOCUMENTATION
  - Generate summary report
  - Include: Building info, climate zone, BPF, baseline EUI, 
    proposed EUI, PI, compliance determination
  - Flag for energy modeler review

Step 5: QA/QC FLAGS
  - Check for anomalous results (e.g., proposed > baseline)
  - Verify all required model runs present
  - Cross-check fuel types match project
```

#### Estimated Time Savings
- Manual process: 40-80 hours (modeling only, not including AI)
- AI-assisted documentation: Saves 2-4 hours of post-modeling calculation and formatting
- **Overall time savings: 5-10% of total credit effort**

### EAp5: Fundamental Refrigerant Management

| Attribute | Detail |
|-----------|--------|
| **Automation Score** | 4 / 5 |
| **Commercial Value** | 4 / 5 |
| **Risk Level** | Low |
| **Documentation Type** | Inventory + Calculation |
| **Recommendation** | **Automate in MVP** |

#### Why This Score
The refrigerant inventory is a **structured data exercise**: equipment list, refrigerant type, GWP values, charge amounts, total GWP calculation. This is highly automatable because:

1. Equipment schedules contain refrigerant type information
2. GWP values are standardized (IPCC/EPA databases)
3. Total GWP is a straightforward calculation
4. HCFC detection is a simple database flag
5. Alternative refrigerant evaluation can be templated

The only non-automatable element is the physical leak check (field testing).

#### Required Inputs

| Input Source | What Design Team Provides | What AI Generates |
|-------------|------------------------------|-------------------|
| Equipment Schedules | Refrigerant-containing equipment list | Complete refrigerant inventory table |
| Equipment Specs | Refrigerant type per equipment | GWP lookup from database |
| Equipment Specs | Refrigerant charge per unit | Total GWP calculation |
| Design Team | New vs. existing equipment designation | HCFC flag report |
| Design Team | GWP>700 refrigerant list | Alternative evaluation template |

#### AI Techniques Applicable

1. **Refrigerant GWP database lookups** -- Automated lookup from EPA/IPCC refrigerant database
2. **Equipment specification parsing** -- Extract refrigerant data from equipment schedules and submittals
3. **Carbon calculation engines** -- Total GWP = sum(refrigerant charge x GWP) across all equipment
4. **Compliance checking** -- Flag HCFC refrigerants, flag GWP>700 refrigerants for alternative evaluation
5. **Document generation** -- Generate complete refrigerant inventory document

#### Automation Blueprint

```
Step 1: EQUIPMENT SCHEDULE PARSING
  - Ingest equipment schedules (PDF, Excel, Revit export)
  - Extract: Equipment ID, Equipment type, Manufacturer, Model
  - Extract: Refrigerant type (R-410A, R-32, R-134a, etc.)
  - Extract: Refrigerant charge per unit (lbs or kg)
  - Extract: Number of units
  - Extract: New vs. existing designation

Step 2: GWP DATABASE LOOKUP
  - Match refrigerant types to GWP values:
    R-410A -> GWP 2,088 (100-year)
    R-32   -> GWP 675
    R-134a -> GWP 1,430
    R-1234yf -> GWP <1
    R-513A -> GWP 573
    [Full database of 200+ refrigerants]
  - Flag any refrigerants not in database for manual lookup

Step 3: INVENTORY CALCULATION
  - Calculate per equipment: Charge (lbs) x GWP = Total GWP
  - Calculate project total: Sum of all equipment GWP
  - Calculate average GWP: Total GWP / Total charge
  - Categorize by equipment type (HVAC, refrigeration, etc.)

Step 4: COMPLIANCE CHECKING
  - HCFC Check: Flag any R-22, R-123, R-401A, etc. (banned in new equipment)
  - GWP>700 Check: Flag equipment requiring alternative evaluation
  - Generate HCFC-free certification statement
  - Generate alternative evaluation requirements list

Step 5: DOCUMENT GENERATION
  - Generate complete refrigerant inventory table:
    | Equipment ID | Type | Qty | Refrigerant | Charge/Unit | 
    | Total Charge | GWP | Total GWP | New/Existing |
  - Generate summary: Total equipment count, Total charge, 
    Total GWP, HCFC status, GWP>700 count
  - Generate alternative evaluation template for flagged equipment

Step 6: LEAK CHECK DOCUMENTATION
  - Generate leak check protocol template
  - Generate repair record template
  - Pre-fill equipment list for field use

Step 7: QUALITY REVIEW
  - Cross-check total charge against typical ranges for building type
  - Verify all equipment captured (compare to MEP schedule count)
  - Flag for professional review
```

### EAc1: Electrification

| Attribute | Detail |
|-----------|--------|
| **Automation Score** | 4 / 5 |
| **Commercial Value** | 5 / 5 |
| **Risk Level** | Low |
| **Documentation Type** | Calculation + Narrative |
| **Recommendation** | **Automate in MVP** |

#### Why This Score
EAc1 is fundamentally a **calculation-based credit** requiring weighted average COP determination for space heating and service water heating equipment. These calculations are highly automatable because:

1. Equipment COP values are published in manufacturer specifications
2. Weighted average calculation is a standard formula
3. Climate zone exclusions are deterministic
4. The compliance determination is pass/fail based on clear thresholds (1.8 COP, 1.3 COP)
5. Core and Shell future-system calculations use defined defaults (COP 1.0)

AI cannot select the equipment (that's design work), but once equipment is specified, AI can fully automate the compliance calculation and documentation.

#### Required Inputs

| Input Source | What Design Team Provides | What AI Generates |
|-------------|------------------------------|-------------------|
| Equipment Schedules | Space heating equipment list with COP ratings | Weighted average COP calculation |
| Equipment Schedules | SWH equipment list with COP or EF ratings | Climate zone exclusion determination |
| Project Data | Climate zone, building type (NC vs C&S) | Compliance determination (points earned) |
| Project Data | Equipment capacities (Btu/hr) | Documentation narrative |
| Design Team | Point-of-use water heater designations | Exclusion justification |

#### AI Techniques Applicable

1. **Equipment specification parsing** -- Extract COP, capacity, and equipment type from schedules
2. **Carbon calculation engines** -- Weighted COP = sum(COP_i x Capacity_i) / Total Capacity
3. **Compliance checking** -- Compare weighted COP to threshold, determine exclusions
4. **Climate zone database** -- Automatic climate zone determination from project location
5. **Document generation** -- Generate compliance narrative with calculations

#### Automation Blueprint

```
Step 1: EQUIPMENT DATA EXTRACTION
  - Parse equipment schedules for heating equipment:
    Space Heating: Type, Manufacturer, Model, COP@47F or COP@17F, 
                   Capacity (Btu/hr), Quantity
    SWH: Type, Manufacturer, Model, COP or Energy Factor,
         Capacity (Btu/hr), Quantity
  - Identify supplemental/auxiliary heating equipment
  - Identify point-of-use water heaters

Step 2: CLIMATE ZONE DETERMINATION
  - Input: Project zip code or lat/long
  - Output: ASHRAE climate zone (0A-8)
  - Apply exclusions:
    - CZ 0-2: Space heating excluded from COP determination
    - All zones: Supplemental low-temp equipment excluded
    - All zones: Point-of-use per ASHRAE 90.1-2022 W05 excluded
    - SWH < 34,000 Btu/hr: Excluded

Step 3: WEIGHTED COP CALCULATION
  For each system (Space Heating, SWH):
    Weighted COP = SUM(COP_i x Capacity_i) / SUM(Capacity_i)
    
  For Core and Shell:
    Future equipment COP = 1.0 (default) or 1.8+ (with documentation)
    Include ALL heating/SWH systems necessary for total building load
    
  Combined calculation (if required):
    Combined COP = SUM(all heating + SWH COP x capacity) / Total capacity

Step 4: COMPLIANCE DETERMINATION
  New Construction:
    Option 1 (No combustion): Combined COP >= 1.8? -> 5 points
    Option 2 Path 1: Space heating COP >= 1.8? -> 2 points
    Option 2 Path 2: SWH COP >= 1.8 OR Solar fraction >= 0.4? -> 1 point
    Option 2 Path 3: No combustion for cooking/process? -> 1 point
  
  Core and Shell:
    Path 1: Combined COP >= 1.8? -> 4 points, >= 1.3? -> 3 points
    Path 2 Case 1: Space heating COP >= 1.8? -> 2 points, >= 1.3? -> 1 point
    Path 2 Case 2: SWH COP >= 1.8 OR Solar fraction >= 0.4? -> 1 point
    Path 3: Scope of work assessment -> 1-2 points
    Option 2: Electrification readiness -> 1 point

Step 5: SOLAR FRACTION CALCULATION (if applicable)
  - Parse solar thermal system specs
  - Calculate solar fraction using f-chart method or simulation output
  - Compare to 0.4 threshold

Step 6: DOCUMENTATION GENERATION
  - Generate compliance narrative including:
    - Equipment inventory table
    - COP determination methodology
    - Climate zone and exclusions applied
    - Weighted COP calculation with formula
    - Compliance determination and points earned
    - Supporting calculations for solar fraction (if applicable)

Step 7: QUALITY REVIEW
  - Verify all equipment captured
  - Check COP values within expected ranges
  - Confirm exclusions properly applied
  - Flag for professional review
```

#### Estimated Time Savings
- Manual calculation + documentation: 4-8 hours
- Automated process: 30 minutes - 1 hour (review)
- **Time savings: 85%**
- **Platinum requirement:** 5 points (NC) / 4 points (C&S) -- this credit is critical for Platinum certification

## Competitive Differentiation Analysis

### What Makes EA Credits Unique for Automation

1. **Deterministic Calculations:** Many EA credits (EAc1, EAc7, EAp5) are pure calculations with clear pass/fail thresholds -- ideal for automation
2. **Standardized Databases:** Refrigerant GWP, ENERGY STAR, ASHRAE tables are all published standards -- perfect for AI integration
3. **High Documentation Burden:** Commissioning plans, decarbonization narratives represent 60-80% of effort -- prime for templating
4. **Mandatory Prerequisites:** 5 of 12 credits are required -- automation provides guaranteed ROI
5. **Platinum Dependencies:** EAc1 (5 pts), EAc3 (8 pts), EAc4 (100% renewable) are critical for Platinum -- automation directly enables higher certification levels

### What Competitors Typically Do

Most existing LEED automation tools focus on:
- **Material credits** (MR category) -- product database lookups
- **Documentation management** -- file storage and tracking
- **Form filling** -- basic data entry automation

**The EA category is underserved** because it requires:
- Engineering domain expertise (energy modeling, refrigeration, HVAC)
- Integration with specialized software (eQuest, EnergyPlus, TRACE)
- Complex calculation engines (carbon, COP, GWP)
- Regulatory database integration (EPA, IPCC, ASHRAE)

This creates a significant competitive moat for a platform that successfully automates EA credits.
